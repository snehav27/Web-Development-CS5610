Experiment 2 :

Responsive Navigation Bar

About Experiment

Responsive web design (RWD) is a web design approach aimed at crafting sites to provide an optimal viewing experience easy reading and navigation with a minimum of resizing, panning, and scrolling—across a wide range of devices (from mobile phones to desktop computer monitors). In this experiment is I have created navigation bar which is responsive according to the devices to view this page .
  1. Car Information
  2. Milage Information
  3. Editor Page
  4. Login Page

View Code Snippet

        <nav role="navigation">

        <ol>

        <li>Car Information</li>

        <li>Milage Information</li>

        <li>Editor Page</li>

        <li>Login Page</li>

        </ol>

Description :

from the above code snippet role ='navigation' to show that content inside the the nav is for navigation , and with the role = 'navigation' , I have manipulated the data in the style tag to show image background and other operations. here in li each attribute indicates the each link in the navigation bar ,how they will be viewed is also maipulated in the style sheet acording to screen size.

CSS Code Snippet

        @media screen and (min-width: 40em) { 

            body {

                display: table;

                
            }

            [role="navigation"] { 

                display: table-caption; 

                background-image: url('../../images/webImages/header1.jpg');

                background-size: 100%;

            }

                [role="navigation"] ol {

                    background-image: url('../../images/webImages/header1.jpg');

                    display: grid;

                }

                    [role="navigation"] ol li {

                       
                        display: table-cell;

                        padding: 4px 2em;

                    }

        }
Description :

@media screen and (min-width: 40em) : will be called according to screen size , when minimum width is 40em then the . screen elements will be resized according to the width of the screen. in 40 em case ol be in grid and li will view as table cell. Till the screen size doesnt reduce below 40 em , same page will be viewed.

         @media screen and (min-width: 10em) {

            body {

                display: table;

                caption-side: top;

            }
            [role="navigation"] {

                display: table-caption;

                background-image: url('../../images/webImages/header1.jpg');

                background-size: 100%;

            }

                [role="navigation"] ol {

                    
                    display: table-row;

                }

                    [role="navigation"] ol li {

                        
                        display: grid;

                        padding: 4px 2em;

                    }

        }

Description :
@media screen and (min-width: 10em) : will be called according to screen size , when minimum width is 10em then the . screen elements will be resized according to the width of the screen. in 10 em case ol be viewes as table row and li will view as grid Till the screen size doesnt reduce below 10 em (practically this will be least width in current mobiles) , same page will be viewed.


Note : We need to always declare minimum of scree size as 40 em (the bigger screen width condition) first else if we place 10 em (the smaller screen width condition) css style first then 10em will overide the 40em and 40 em case will not work.
