About Experiment

This Experiment is continuation of the experiments of Responsive experiments :
Tables are part of every website but the are designed in such a way they cannot be modified with width or height , Even if we do it may turn ugly which no user or client would love to see. We can follow different approch which is possible through CSS.Current table view model can removed and data in the table can be interpreted as sheet of information as shown in the experiment.

View Code :

    <table>
            <thead> 

            <tr>

            <th>First Name</th>

            <th>Last Name</th>

            <th>Job Title</th>

            <th>Favorite Color</th>

            <th>Dream Vacation City</th>

            </tr>

        </thead>

    <tbody>

             <tr>

             <td>Sneha</td>

             <td>Vankireddy</td>

            <td>Software Engineer</td>

            <td>Pink</td>

             <td>Paris</td>

            </tr>

           </table>
Description :

This table view is same as any normal table , but the trick is done in the CSS file to convert the details in the table as sheet of information

CSS code snippet : 

 
         @media only screen and (max-width: 760px), (min-device-width: 768px) and (max-device-width: 1024px) {

          
            table, thead, tbody, th, td, tr {

                display: block;

            }

             
                thead tr {

                    position: absolute;

                    top: -9999px;

                    left: -9999px;

                }
            tr {
                border: 1px solid #ccc;

            }
            td {               
                border: none;

                border-bottom: 1px solid #eee;

                position: relative;

                padding-left: 50%;

            }
                td:before {
  
                    position: absolute;
                
                    top: 6px;

                    left: 6px;

                    width: 45%;

                    padding-right: 10px;

                    white-space: nowrap;

                }

                 td:nth-of-type(1):before {

                    content: "First Name";

                }

                td:nth-of-type(2):before {

                    content: "Last Name";

                }

                td:nth-of-type(3):before {

                    content: "Job Title";

                }
                td:nth-of-type(4):before {

                    content: "Favorite Color";

                }

                td:nth-of-type(5):before {

                    content: "Dream Vacation City";

                }

        }
Description :

As we know from my previour experiments CSS @media queries is to detect if the screen is smaller than the maximum squishitude of our table. If it is, we're going to reformat the table. In the CSS file we are forcing the table to not behave like table any more and blocking it also hidding headers , every table-related element to be block-level. Here we are making table row to become table itself and making the table coloumn act as table row. which will alter the view when data is viewed in the small screen. Also we need to mention before elements of td element which will be staring from 0,1 ,2 ... element when data changes .
